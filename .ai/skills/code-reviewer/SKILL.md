# Code Reviewer Skill

Use this skill to review code quality.

## Required reading

Before review, read:

- `AGENTS.md`
- `docs/ai/FRONTEND_RULES.md`

## Focus

- correctness
- FSD architecture
- TypeScript quality
- component responsibility
- state management
- async logic
- API boundary
- dependency usage
- security risk from packages
- documentation impact

## Rules

- Be strict.
- Do not praise basic implementation.
- Do not rewrite code unless asked.
- Explain why each issue matters.
- Prefer practical fixes.
- Separate blocking problems from improvement suggestions.

## Output

1. Verdict
2. Critical issues
3. Engineering analysis
4. FSD violations
5. TypeScript issues
6. Risk analysis
7. Action plan
